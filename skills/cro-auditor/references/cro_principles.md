# Principes CRO - Guide de Référence Audit

## 1. Les 11 Erreurs Classiques LP/Sites

### 1.1 Page déconnectée des ads (Scent Trail brisé)

**Description**
La page d'arrivée ne poursuit pas la promesse faite dans l'annonce. Rupture entre le contexte publicitaire et le contenu page.

**Types de déconnexions**
- Message match : annonce dit X, page dit Y
- Visual match : style/couleurs différents
- Tone match : ton/langage discordant
- Offer match : offre modifiée ou absente
- Audience match : page cible un autre persona

**Impact sur la conversion**
- Augmente le bounce rate (+20-40%)
- Diminue le scroll depth et engagement
- Réduit la CVR de 15-30%
- Augmente le coût par lead

**Checklist audit**
- [ ] Copier les headlines d'ads et vérifier si elles apparaissent dans la page
- [ ] Comparer les couleurs primaires (ads vs page)
- [ ] Vérifier que l'offer est explicite et identique
- [ ] Contrôler que le target audience correspond
- [ ] Tester les landing pages via mobile (comme les utilisateurs)

**Comment corriger**
1. Utiliser les mêmes headlines/sous-headlines que l'ad
2. Reprendre la palette de couleurs de l'ad en hero
3. Adapter le tone/langage au segment d'audience
4. Afficher l'offre exacte dès le hero (discount, bonus, etc.)
5. Créer des variantes de page pour différents segments d'audience

---

### 1.2 Manque de Social Proof

**Description**
Absence ou mauvais placement de preuves sociales (avis, logos, chiffres, témoignages, vidéos).

**Hiérarchie des preuves (efficacité décroissante)**
1. Vidéos testimoniales authentiques
2. Avis vérifiés (Google, Trustpilot, plateforme native)
3. Screenshots avis/messages clients
4. Chiffres/statistiques (X clients, Y% satisfaction)
5. Logos clients/partenaires
6. Badges de certification/trust

**Impact sur la conversion**
- CVR -20-35% sans social proof
- Réduit la confiance et crédibilité
- Augmente les abandon de panier

**Checklist audit**
- [ ] Compter le nombre d'éléments de social proof
- [ ] Vérifier la hiérarchie utilisée
- [ ] S'assurer que les avis sont vérifiés/authentiques
- [ ] Valider la lisibilité et accessibilité (petite police?)
- [ ] Contrôler la placement (visible + itérée)

**Comment corriger**
1. Prioriser les vidéos testimoniales (enregistrements clients)
2. Afficher des avis vérifiés (Google, Trustpilot, site tiers)
3. Intégrer 2-3 éléments de social proof minimum dans hero + middle
4. Ajouter des chiffres concrets (clients satisfaits, taux de conversion, etc.)
5. Utiliser des logos clients prestigieux
6. Itérer la social proof (pas juste en haut)

---

### 1.3 Frictions dans le tunnel

**Description**
Obstacles créant de la résistance chez l'utilisateur durant son parcours. 5 types de friction à évaluer.

**Les 5 types de friction**
1. **Friction cognitive** : message confus, trop d'options, jargon, décision difficile
2. **Friction visuelle** : UI désorganisée, couleurs cassées, typo mauvaise, manque de contraste
3. **Friction de navigation** : formulaires longs, étapes confuses, boutons peu clairs
4. **Friction de confiance** : pas de garantie, données perso demandées trop tôt, absence de RGPD
5. **Friction technique** : bugs, erreurs de formulaire, lenteur, crashes mobile

**Concept IPP (Interaction Path Psychology)**
Chaque interaction doit être claire, rapide et récompensante. Chaque clic/tap = micro-engagement.

**Impact sur la conversion**
- Augmente le bounce rate de 10-50% par friction majeure
- Augmente le cart abandonment de 20%+
- Réduit la CVR de 25-45%

**Checklist audit**
- [ ] Tester le parcours utilisateur complet (desktop + mobile)
- [ ] Évaluer chaque type de friction
- [ ] Compter les clics requis pour convertir
- [ ] Identifier les points de drop-off
- [ ] Tester sur 4G lent (throttling)

**Comment corriger**
1. Simplifier le copy (max 10 mots par headline)
2. Réduire les étapes formulaire (chunked form > long form)
3. Utiliser des CTA clairs et orientés bénéfice
4. Assurer une hiérarchie visuelle nette
5. Tester les formulaires exhaustivement
6. Ajouter des indicateurs de confiance (badges, garanties)

---

### 1.4 Hero mal optimisé

**Description**
La section hero (premier écran) ne capte pas l'attention ou ne transmet pas la proposition de valeur clairement.

**Test des 5 secondes**
Utilisateur ne voit le hero que ~5 sec en scrolling. Doit comprendre la proposition ET avoir envie de continuer.

**Éléments critiques hero**
- Headline claire (benefit-driven, pas tagline)
- Sous-titre/sous-headline pour clarifier
- CTA visible et contrastant
- Image/visuel produit pertinent
- Micro-réassurance (garanti, sans engagement, etc.)

**Impact sur la conversion**
- CVR -40-60% sans hero optimisé
- Bounce rate augmente si message pas clair
- Scroll depth diminue

**Checklist audit**
- [ ] Appliquer le test des 5 secondes (montrer hero 5s puis demander proposition de valeur)
- [ ] Vérifier que le headline est un benefit, pas un tagline
- [ ] S'assurer que le CTA est visible et cliquable
- [ ] Valider que le visuel correspond à la promesse
- [ ] Vérifier la micro-réassurance (ex: "essai gratuit", "sans CB", etc.)

**Comment corriger**
1. Réécrire headline sur format : [Bénéfice] pour [Audience] qui [Problème]
2. Ajouter un sous-titre pour clarifier l'offre
3. Placer CTA en haut (bouton sticky mobile recommandé)
4. Utiliser image produit ou démo visuelle
5. Ajouter un élément de micro-réassurance (badge, garantie, urgence)

---

### 1.5 Mauvaise hiérarchisation (F-pattern / Z-pattern)

**Description**
Organisation visuelle ne guide pas le regard naturellement. Utilisateur perd en lisibilité et compréhension.

**Patterns de lecture**
- **F-pattern** : top-left → droite → scan vertical → répète. Bon pour article/texte.
- **Z-pattern** : top-left → top-right → diagonal bas → bas-gauche → bas-droit. Bon pour vendre.
- Un message majeur par section (éviter le fouillis)

**Hiérarchie visuelle efficace**
1. Headline (plus grand, contraste fort)
2. Sous-headline (plus petit, accent couleur)
3. Body copy (plus petit encore, texte gris)
4. CTA (couleur primaire, taille large)

**Impact sur la conversion**
- CVR -15-30% si hiérarchie mauvaise
- Augmente le bounce rate (utilisateur s'y perd)
- Réduit le temps passé sur page

**Checklist audit**
- [ ] Tracer le F ou Z pattern sur la page
- [ ] Vérifier que les titres sont scanables
- [ ] Valider un seul message principal par section
- [ ] S'assurer du contraste texte/fond
- [ ] Tester que les titres sont lisibles à distance (15cm écran)

**Comment corriger**
1. Organiser par sections claires (1 idée = 1 section)
2. Utiliser tailles de police : H1 > H2 > H3 > Body
3. Ajouter whitespace entre sections
4. Utiliser la couleur pour mettre en avant le CTA
5. Tester sur mobile : titres doivent être scanables au pouce

---

### 1.6 Bugs techniques

**Description**
Erreurs de fonctionnement (formulaires, boutons, vidéos, formulaires) impactant la conversion.

**Checklist QA technique**
- [ ] Tous les CTA sont cliquables et fonctionnels
- [ ] Formulaires se soumettent correctement
- [ ] Liens internes/externes ouvrent correctement
- [ ] Pas de 404 ou redirections cassées
- [ ] Vidéos se lancent et jouent
- [ ] Optimisation des images (pas blurrées)
- [ ] PDF/documents téléchargeables
- [ ] Chat/formulaires de contact fonctionnels

**Cross-browser testing**
- Chrome (+ Android via Chrome mobile)
- Safari (+ iOS via Safari)
- Firefox
- Edge

**Impact sur la conversion**
- CVR -100% si CTA ne fonctionne pas
- Bounce rate +30-50% si bugs visibles
- Perte de crédibilité utilisateur

**Checklist audit**
- [ ] Tester sur 3-4 navigateurs
- [ ] Tester sur mobile (throttling 4G)
- [ ] Soumettre un test formulaire
- [ ] Vérifier les emails de confirmation
- [ ] Tester les redirects post-conversion

**Comment corriger**
1. Créer checklist QA par page type
2. Automatiser les tests cross-browser (BrowserStack, etc.)
3. Tester les formulaires avec données réelles
4. Ajouter des validations claires (error messages)
5. Configurer des alertes d'erreurs (Sentry, etc.)

### 1.7 Pas Mobile-First (85-95% du trafic)

**Description**
Page optimisée desktop mais non-fonctionnelle/mauvaise UX mobile (85-95% du trafic sur mobile pour paid social).

**Mobile-First priorities**
- Thumb zone : boutons/CTA dans zone accessible au pouce (bas de l'écran)
- CTA sticky en haut/bas pour accessibilité
- Images format portrait (pas paysage)
- Texte responsive (grandeur adaptée à l'écran)
- Pas de pop-ups bloquantes non-closable
- Pas de autoplay vidéo avec son

**Impact sur la conversion**
- CVR -30-50% si page non-mobile optimisée
- Bounce rate +40-60%
- Google pénalise les pages non-mobile friendly

**Checklist audit**
- [ ] Tester sur vrais devices (iPhone 12-14, Samsung Galaxy)
- [ ] Vérifier que tous les CTA sont accessibles au pouce
- [ ] Valider que les images s'affichent bien en portrait
- [ ] Tester les formulaires sur mobile (padding entre champs)
- [ ] Vérifier que la navigation n'est pas bloquante

**Comment corriger**
1. Utiliser framework mobile-first (Bootstrap, Tailwind)
2. Mettre CTA sticky en haut/bas (sauf top 20% de la page)
3. Adapter le design : une colonne, plus d'espace vertical
4. Compresser les images pour mobile (lazy loading)
5. Désactiver pop-ups bloquantes sur mobile

---

### 1.8 Trop de liens de sortie (Menu, Footer)

**Description**
Navigation, menu ou footer offrant trop d'alternatives → utilisateur distrait → abandonne la page.

**Ratio d'attention 1:1**
Par CTA principal, max 1 lien de sortie. Exemple : si 1 CTA de conversion, max 1 lien alternatif (ex: FAQ, contact).

**Éléments à limiter**
- Menu navigation (caché en hamburger mobile)
- Liens footer (max 3-4 catégories)
- Liens internes (pas de navigation vers autres produits/pages)
- Liens externes (au strict minimum)

**Impact sur la conversion**
- CVR -20-40% avec menu/footer chargés
- Augmente les abandons vers autres pages
- Réduit la concentration utilisateur

**Checklist audit**
- [ ] Compter les liens de sortie hors CTA principal
- [ ] Vérifier si menu/nav visible sur mobile
- [ ] Analyser le footer (trop de liens?)
- [ ] Identifier les liens alternatifs distrayants
- [ ] Comparer avec pages haute-CVR (reference)

**Comment corriger**
1. Masquer le menu hamburger sur mobile (ou le rendre transparent)
2. Réduire le footer aux infos essentielles (pas de liens)
3. Supprimer les liens internes distrayants
4. Conserver max 3-4 liens alternatifs (FAQ, Contact, Charte)
5. Tester sans footer → observer impact sur CVR

---

### 1.9 Infos principales sous le fold

**Description**
Les éléments critiques (proposition de valeur, CTA, social proof) ne sont pas visibles "above the fold" (1er écran).

**Above the fold must-haves**
- Headline de la proposition de valeur
- Sous-titre/clarification (ce qu'on vend)
- CTA principal (visible et cliquable)
- Élément visuel (image produit ou démo)
- Optionnel : logo/award/social proof minime

**Benchmark**
Minimum : 50% de l'écran doit être rempli d'info pertinente (pas vide/image vague).

**Impact sur la conversion**
- CVR -25-40% si essentiels sont below fold
- Bounce rate +30% si proposition pas claire immédiatement
- Scroll depth réduit

**Checklist audit**
- [ ] Faire screenshot du fold (largeur mobile 375px, hauteur ~667px)
- [ ] Identifier les éléments visibles dans le fold
- [ ] Vérifier que headline + CTA sont accessibles
- [ ] S'assurer que visuel est pertinent (pas juste décoration)
- [ ] Valider que 50%+ de l'écran est "informatif"

**Comment corriger**
1. Repositionner headline/CTA plus haut (dans le fold)
2. Réduire la taille du héro (moins de vertical scrolling requis)
3. Utiliser image plus petite ou utiliser le whitespace
4. Prioriser texte par rapport à image (si conflit d'espace)
5. Tester le fold sur mobile (le plus critique)

---

### 1.10 Mauvaise vitesse de chargement

**Description**
Page lente (>3s) → utilisateurs quittent → CVR réduite. Google pénalise aussi.

**Core Web Vitals (CWV) - Benchmarks**
- **LCP (Largest Contentful Paint)** : <2.5s (bon), 2.5-4s (à améliorer), >4s (mauvais)
- **FID/INP (Interaction)** : <100ms (bon), 100-300ms (à améliorer), >300ms (mauvais)
- **CLS (Layout Shift)** : <0.1 (bon), 0.1-0.25 (à améliorer), >0.25 (mauvais)

**Optimisations courantes**
- Lazy loading images
- Compression images (Webp, format adapté)
- Minification CSS/JS
- CDN pour assets
- Caching browser
- Réduction du JS (async/defer)

**Impact sur la conversion**
- CVR -7% par seconde supplémentaire (après 3s)
- Bounce rate +25-50% si chargement >5s
- Google baisse le ranking

**Checklist audit**
- [ ] Tester PageSpeed Insights (Google)
- [ ] Vérifier LCP/FID/CLS
- [ ] Utiliser GTmetrix ou WebPageTest
- [ ] Tester sur 4G throttling
- [ ] Identifier images/ressources lourdes

**Comment corriger**
1. Compresser et optimiser images (TinyPNG, Squoosh)
2. Activer lazy loading sur images
3. Utiliser CDN (Cloudflare, AWS CloudFront)
4. Minimiser CSS/JS externe
5. Mettre en cache les ressources statiques
6. Tester et itérer (chaque amélioration compte)

---

### 1.11 Pas de tracking/analytics

**Description**
Absence ou mauvaise configuration du tracking → impossible de mesurer CVR, source de traffic, comportement utilisateur.

**Éléments de tracking essentiels**
- Pixel conversion (Facebook, Google Ads, LinkedIn)
- CAPI (Conversion API) pour meilleure attribution
- Events personnalisés (view_item, add_to_cart, purchase, etc.)
- UTM parameters (source, medium, campaign, content)
- Heatmap/Session recording (Hotjar, Smartlook)
- Analytics (GA4, configuré correctement)

**Checklist technique**
- [ ] Pixel ads configuré et testé (Facebook, Google)
- [ ] CAPI connecté si applicable
- [ ] GA4 tracking ID installé
- [ ] Events principaux configurés
- [ ] UTM parameters dans tous les liens ads
- [ ] Heatmap/session recording activé
- [ ] Tester conversion end-to-end

**Impact sur la conversion**
- Impossible d'optimiser sans data
- Attribution incorrecte du ROI
- Décisions d'optimization basées sur suppositions

**Checklist audit**
- [ ] Ouvrir tag manager/console pour vérifier pixels
- [ ] Tester une conversion (vérifier confirmation)
- [ ] Vérifier que la conversion remonte en time réel (Ads Manager, GA4)
- [ ] Valider les UTM parameters en logs
- [ ] Vérifier que heatmap enregistre sessions

**Comment corriger**
1. Installer Google Tag Manager + GA4
2. Configurer le pixel Ads (Facebook, Google)
3. Configurer CAPI si budget permet
4. Ajouter events personnalisés (JavaScript)
5. Ajouter UTM parameters dans tous les liens
6. Mettre en place heatmap (Hotjar gratuit)
7. Valider tracking en temps réel

## 2. Principes CRO Fondamentaux

### 2.1 Scent Trail (Piste Olfactive)

**Définition**
Continuité cohérente entre la promesse publicitaire et l'expérience page. L'utilisateur doit sentir qu'il est au bon endroit.

**4 dimensions du scent trail**
1. **Message match** : même proposition de valeur
2. **Visual match** : palette couleur, style d'images
3. **Tone match** : langage, ton de voix cohérent
4. **Audience match** : page parle au même persona que l'ad

**Audit rapide**
- Copier texte d'ad → chercher sur page
- Comparer 3 couleurs primaires (ad vs page)
- Vérifier que l'offer est explicite

**Impact**
Chaque rupture de scent trail = -15% CVR en moyenne.

---

### 2.2 Ratio d'Attention 1:1

**Définition**
Pour chaque CTA principal, max 1 lien alternatif (distraction).

**Règle simple**
- 1 CTA principal (conversion) → max 1 lien alternatif (FAQ, Contact, etc.)
- 2 CTAs → max 2 liens alternatifs
- etc.

**Éviter**
- Menus de navigation visibles
- Footers chargés de liens
- Liens internes vers autres produits
- Trop de choix = paralysie décision

**Impact**
Chaque lien supplémentaire = -8-12% CVR.

---

### 2.3 Test des 5 Secondes

**Définition**
Un utilisateur en scrolling voit la page ~5 secondes max avant de décider de rester/partir.

**Application**
1. Montrer le hero pendant 5 secondes
2. Demander : "Qu'est-ce qu'on vend? À qui? Quel est le bénéfice?"
3. Si l'utilisateur n'a pas la réponse claire → fix le hero

**Éléments clés**
- Headline clique (bénéfice, pas tagline)
- Image pertinente
- CTA visible
- Pas de confusion

**Benchmark**
Minimum 80% des testeurs doivent pouvoir répondre correctement.

---

### 2.4 Hiérarchie de la Preuve Sociale

**Ordre d'efficacité (décroissant)**

| Rang | Type | Efficacité | Exemple |
|------|------|-----------|---------|
| 1 | Vidéo testimoniale | ★★★★★ | Client raconte son histoire |
| 2 | Avis vérifiés | ★★★★★ | Google Reviews, Trustpilot |
| 3 | Screenshots avis | ★★★★ | Captures de commentaires clients |
| 4 | Chiffres/stats | ★★★★ | "10,000 clients satisfaits" |
| 5 | Logos clients | ★★★ | "Utilisé par Google, Microsoft..." |
| 6 | Badges/certifications | ★★ | "Certifié XYZ", "Award 2024" |
| 7 | Testimonial simple | ★ | Texte anonyme sans vérification |

**Audit rapide**
- Quelle est la preuve la plus évidente? (doit être vidéo ou avis vérifiés)
- Est-elle itérée? (doit apparaître plusieurs fois)
- Est-elle authentique? (vérifier la source)

### 2.5 Les 5 Types de Friction

#### 1. Friction Cognitive
Message confus, trop d'options, jargon, décision difficile.

**Signes d'alerte**
- Headline >15 mots
- Trop de sous-sections
- Jargon technique non-expliqué
- Multiple value propositions (au lieu d'une)

**Fix**
- Simplifier à 1 idée = 1 section
- Max 10 mots par headline
- Traduire jargon en langage courant

#### 2. Friction Visuelle
UI désorganisée, couleurs cassées, typo mauvaise, manque de contraste.

**Signes d'alerte**
- Texte illisible (petit, mauvais contraste)
- Couleurs discordantes
- Trop de polices différentes
- Images flous/bruts

**Fix**
- Max 2-3 polices (heading + body)
- Contraste min 4.5:1 (WCAG AA)
- Cohérence couleurs (palette de 3-5 couleurs)
- Images optimisées (Webp, haute qualité)

#### 3. Friction de Navigation
Formulaires longs, étapes confuses, boutons peu clairs.

**Signes d'alerte**
- Formulaire >7 champs
- Pas d'indication d'étapes
- Boutons peu visibles
- Erreurs de formulaire peu claires

**Fix**
- Diviser formulaire (progressive profiling)
- Ajouter progress bar
- CTA contrastant + texte orienté bénéfice
- Messages d'erreur clairs et constructifs

#### 4. Friction de Confiance
Pas de garantie, données perso demandées trop tôt, absence de RGPD.

**Signes d'alerte**
- Pas de garantie/refund promise
- Email requis dès le départ (sans justification)
- Pas de badge de sécurité
- Pas de lien RGPD/Privacy
- Paiement demandé sans preuve sociale

**Fix**
- Ajouter "Satisfait ou remboursé" ou garantie similaire
- Demander email/données progressivement
- Afficher badges sécurité (SSL, certifications)
- Lien Privacy Policy visible
- Social proof avant demande de paiement

#### 5. Friction Technique
Bugs, erreurs formulaire, lenteur, crashes mobile.

**Signes d'alerte**
- Formulaires non-fonctionnels
- CTA cassés/non-cliquables
- Erreurs console (F12)
- Chargement lent (>3s)
- Bugs mobile (responsive broken)

**Fix**
- QA complète (tous navigateurs)
- Optimisation vitesse
- Tests formulaires
- Responsive design testing

### 2.6 F-Pattern et Z-Pattern

#### F-Pattern (lecture linéaire)

Utilisé par défaut pour contenu textuel.

```
Top-left ➜ Top-right
   ↓
Mid-left ➜ Mid-right
   ↓
Bottom-left ➜ Bottom-right
```

**Quand utiliser**
- Article/blog
- Description produit longue
- Comparatif détaillé

#### Z-Pattern (vente)

Utilisé pour capturer et convertir.

```
Top-left ➜ Top-right
    ↘
  Center
    ↙
Bottom-left ➜ Bottom-right
```

**Quand utiliser**
- Landing page
- Hero section
- Call-to-action flow

**Application pratique**
- Placer le logo/heading en top-left
- Proposition de valeur en top-right
- Visuel/bénéfice au center
- CTA en bottom-right

### 2.7 Mobile-First Design Principles

#### Priorités Mobile (85-95% du trafic)

1. **Thumb Zone**
   - CTA doit être accessible au pouce (bas de l'écran)
   - Boutons min 48x48px
   - Padding entre éléments

2. **One-Column Layout**
   - Pas de grille complexe
   - Contenu vertical (scroll, pas scroll latéral)
   - Fullwidth sections

3. **Sticky CTA**
   - CTA collant en haut ou bas (sauf top 20%)
   - Reste visible en scrollant
   - Alternativement : CTA répétée chaque 3-4 sections

4. **Portrait Images**
   - Images au format 4:5 ou 9:16 (vertical)
   - Pas d'images paysage (horizontal)
   - Héro image full-width

5. **No Autoplay**
   - Vidéos ne se lancent pas automatiquement
   - Pas de son autoplay (illegal sur mobile)
   - Play button visible

6. **Form Optimization**
   - Max 3-5 champs par écran
   - Textarea assez grand (touch-friendly)
   - Suggest/autofill activés

7. **No Pop-ups**
   - Pas de modales bloquantes
   - Si pop-up : close button évident
   - Préférer : sticky bar, inline form

---

### 2.8 Above the Fold Best Practices

**Zone critique** : Premiers ~600-800px (mobile) ou 800-1000px (desktop).

**Must-haves dans le fold**
- ✓ Headline (bénéfice, pas tagline)
- ✓ Sous-titre (clarifier l'offer)
- ✓ CTA principal (bouton visible)
- ✓ Visuel pertinent (produit, démo, ou image lifestyle)
- ✓ Micro-réassurance (optionnel : badge, "essai gratuit", etc.)

**À éviter dans le fold**
- ✗ Espace vide (wasted space)
- ✗ Images génériques (non pertinentes)
- ✗ Trop de texte
- ✗ Menu de navigation
- ✗ Footer
- ✗ Plusieurs propositions de valeur différentes

**Audit**
Faire screenshot viewport mobile (375x667) : minimum 50% doit être informatif et pertinent.

### 2.9 Benefit Laddering

**Concept**
Escalader progressivement de Features (QUOI) → Avantages (COMMENT) → Bénéfices (POURQUOI) → Transformation (QUI SERAS-TU).

**Exemple**
```
Feature       : "30 minuteurs prédéfinis"
Avantage      : "Pas besoin de configurer"
Bénéfice      : "Gagne 5 minutes chaque jour"
Transformation: "Deviens quelqu'un de productif et organisé"
```

**Application en copy**
- **Hero** : Bénéfice + Transformation
- **Section 1** : Bénéfice (le plus important)
- **Section 2-3** : Avantages (comfort, délai, prix, etc.)
- **Section 4+** : Features (détails techniques)

**Audit rapide**
Lire chaque section : peut-on répondre "Et alors?" Une fois, deux fois, trois fois?
- Section dit "Feature X" → et alors? (doit dire l'avantage)
- Avantage → et alors? (doit dire le bénéfice)
- Bénéfice → et alors? (doit résoudre un problème émotionnel)

### 2.10 Niveaux de Conscience (Schwartz)

**Modèle des 5 niveaux de conscience client**

| Niveau | Description | Copy Strategy | Exemple |
|--------|-------------|----------------|---------|
| **Unaware** | Client ignore le problème | Créer la prise de conscience du problème | "Tu ignores peut-être que..." |
| **Problem Aware** | Connaît le problème mais pas la solution | Montrer que des solutions existent | "Heureusement, il existe..." |
| **Solution Aware** | Sait qu'il existe des solutions | Comparer les solutions | "Comparé aux alternatives..." |
| **Product Aware** | Connaît le produit/marque | Convaincre que c'est le meilleur | "Voici pourquoi on est meilleur" |
| **Most Aware** | Prêt à acheter | Faire la promotion, urgence, objections | "Offre limitée, clique ici" |

**Application**
- **Ads cold audience** : niveaux 1-2 (créer awareness)
- **Ads warm audience** : niveaux 2-3 (solutions)
- **Landing page** : niveau du audience segment
- **Retargeting** : niveau 4-5 (conversion immédiate)

**Audit**
Identifier le niveau de conscience cible. La page parle-t-elle au bon niveau?

### 2.11 CTA Best Practices

#### Orientation Bénéfice (pas action)

**Mauvais**
- "Soumettre"
- "Envoyer"
- "Cliquer ici"

**Bon**
- "Démarrer ma démo gratuite"
- "Obtenir mon audit gratuit"
- "Rejoindre les 1000+ entrepreneurs"

#### Répétition

- Minimum 3-5 CTAs par page (hero + fin chaque section + sticky)
- Format : même CTA texte (test A/B possible)
- Placement : hero, middle, bottom

#### Contraste

- Couleur primaire (distinct du reste)
- Taille : suffisamment grande pour être cliquable (48x48px min)
- État hover/active visible

#### Urgence (optionnel mais efficace)

- "Offre limitée : 100 places"
- "Expire dans 3 jours"
- "Rejoins les 5000+ déjà inscrits"
- "Dernier jour à ce tarif"

#### Audit CTA

- [ ] Texte orienté bénéfice?
- [ ] Répété au minimum 3-5 fois?
- [ ] Couleur contrastante?
- [ ] Taille suffisante (mobile)?
- [ ] État hover/active visible?
- [ ] Pas de jargon technique?
- [ ] Crée-t-il une micro-engagement?

## 3. Métriques CRO Essentielles

### 3.1 Principales KPIs

| KPI | Formule | Benchmark | Bon | Excellent |
|-----|---------|-----------|------|-----------|
| **CVR** (Conversion Rate) | Conversions / Visitors | 0.5-3% | 2-4% | 5%+ |
| **RPV** (Revenue Per Visitor) | Revenue / Visitors | $0.5-5 | $5-20 | $20+ |
| **AOV** (Average Order Value) | Revenue / Orders | $30-200 | $100-300 | $300+ |
| **Bounce Rate** | Single-page sessions / All sessions | 40-60% | <40% | <30% |
| **Scroll Depth** | Avg. % scrolled | 50-70% | 70-85% | 85%+ |
| **CTA CTR** | CTA clicks / Total visitors | 5-15% | 15-25% | 25%+ |
| **Cart Abandonment** | Carts created / Purchases | 60-85% | <60% | <40% |
| **Form Completion Rate** | Forms submitted / Forms opened | 30-50% | 50-70% | 70%+ |

### 3.2 Benchmarks par Industrie

#### E-commerce
- CVR attendue : 1-3%
- AOV : $50-200
- Bounce Rate : <50%

#### SaaS
- CVR attendue : 2-5%
- Lead quality : email valid + engagement
- Trial to Paid : 5-15%

#### Lead Generation (B2B)
- CVR attendue : 5-15%
- Form Completion : >50%
- Lead quality : data valides

#### Advertorial
- CVR attendue : 1-5% (lead) ou 3-10% (view-through)
- Scroll depth : >70%

#### E-learning
- CVR attendue : 2-8% (course signup)
- Completion rate : >60%

## 5. Checklist Audit Rapide

**Template audit 30 min**

### Pre-audit (5 min)
- [ ] Identifier le type de page
- [ ] Vérifier les objectifs (conversion, lead, etc.)
- [ ] Documenter le contexte (audience, offer, channel)

### Scent Trail (5 min)
- [ ] Comparer ads vs page (message, visuals, tone, offer)
- [ ] Noter les ruptures de scent trail

### Hero & Copy (5 min)
- [ ] Test des 5 secondes
- [ ] Vérifier benefit laddering
- [ ] Évaluer social proof presence

### UX/Friction (5 min)
- [ ] Tester CTA (tous fonctionnels?)
- [ ] Évaluer liens de sortie (ratio 1:1?)
- [ ] Mobile responsiveness check

### Vitesse & Tech (5 min)
- [ ] PageSpeed Insights score
- [ ] Vérifier tracking (pixel, GA4, CAPI)
- [ ] Test de formulaire (si applicable)

### Scoring
Noter chaque section de 0-10. Moyenne = score global.
- 0-50 : Critical issues
- 50-70 : Opportunités d'amélioration
- 70-85 : Bon, optimisable
- 85-95 : Très bon
- 95+ : Excellence
