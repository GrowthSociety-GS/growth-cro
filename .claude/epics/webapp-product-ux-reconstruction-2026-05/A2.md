## Description

Produire le document canonique `WEBAPP_TARGET_IA_2026-05.md` qui fixe **l'information architecture cible** de la webapp : 5 espaces principaux (Command Center, Clients, Audits & Recos, GSG Studio, Advanced Intelligence) + 2 utilitaires (Doctrine, Settings), avec navigation workflow-first, mapping ancien → nouveau, et nav schema sidebar+topbar+breadcrumbs.

**Implementation notes** :

5 espaces proposés (à valider) :

1. **Command Center** (= home `/`) — vue exécutive, 4 zones (Today/Urgent · Fleet Health · Recent Runs · Next Best Actions)
2. **Clients** (= `/clients` + `/clients/[slug]` workspace) — fiche client comme workspace central, tabs secondaires (Audits, Recos, GSG, Reality, Scent, History)
3. **Audits & Recos** (= `/audits` + `/recos` + `/audits/[c]/[id]` fusionnés en workflow) — workflow audit→reco→opportunity board
4. **GSG Studio** (= `/gsg/studio` + `/gsg/runs` + `/gsg/runs/[id]` + `/gsg/design-grammar/[client]`) — vrai studio E2E
5. **Advanced Intelligence** (= `/reality` + `/geo` + `/learning` + `/experiments` + `/scent` groupés sous un menu) — modules avec Maturity Model honnête

+ 2 utilitaires :
- **Doctrine** (= `/doctrine`) — référence pédagogique, hors workflow
- **Settings** (= `/settings`) — org, team, billing

Sidebar uniforme sur **toutes** routes sauf public (`/login`, `/privacy`, `/terms`).

Routes archivées de la nav : `/audit-gads`, `/audit-meta` (cf. FR-25).
