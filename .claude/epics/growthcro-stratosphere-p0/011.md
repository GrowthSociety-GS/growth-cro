# Task: Verdict Gate aggregator in multi-judge

## Description

Créer le **Verdict Gate** dans `moteur_multi_judge/orchestrator.py` qui force `verdict_tier = "🔴 Non shippable"` + `final_score_pct = 0` si **n'importe lequel** des signaux bloquants suivants = FAIL :

- `impeccable_report.passed == False` (impeccable_qa heuristic)
- `doctrine_judge.killer_rules_violated == True` (V3.2.1 killer rules)
- `implementation_judge.impl_penalty > 10pp` (bugs runtime majeurs)
- Tout exit code != 0 parmi les 7 scripts `check_gsg_*.py`
- Exit code != 0 de `SCHEMA/validate_all.py`

Le multi-judge calcule actuellement `verdict_tier` (🚀 Stratosphérique ≥92%, 🌟 Exceptionnel 85-92%, ✅ Bon 75-85%, etc.) **sans jamais consulter ces gates**. Un GSG peut être noté 92% Exceptionnel alors qu'un killer rule a violé ou un claim n'a pas de source.

Source : `AUDIT_DECISION_DOSSIER_2026-05-16` §6 P0.1 + §7 ISSUE-P0-01 + §10.2 Tableau gates.
