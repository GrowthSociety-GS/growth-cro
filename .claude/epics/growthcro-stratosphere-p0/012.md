# Task: ClaimsSourceGate HTML parser + wire mode_1

## Description

Créer le **Claims Source Gate** : un module qui parse le HTML rendu par le GSG, extrait tous les "claims" (chiffres, testimonials, logos, proof statements), et vérifie que **chacun** a un attribut `data-evidence-id` pointant vers une entrée valide de `evidence_ledger.json`. **Refuse** (raise) si un claim n'a pas de source.

Cela ferme la dernière fuite : `impeccable_qa.internal_provenance_leak` détecte certains cas mais c'est juste un avertissement (cf. audit gates §10.2). Aujourd'hui un testimonial sans source peut être rendu sans aucun gate bloquant.

Action complémentaire : étendre le schema `evidence_ledger.json` pour enforce `source_type ∈ {vision, dom, api_external, rule_deterministic}` strict — **jamais** `llm_classifier` seul (cf. P0.2 du plan).

Source : `AUDIT_DECISION_DOSSIER_2026-05-16` §6 P0.2 + §7 ISSUE-P0-02.

## Acceptance Criteria

### Schema extension

- [ ] Modifier `skills/site-capture/scripts/evidence_ledger.py` (ou modèle Pydantic équivalent) :
  - Enum strict `EvidenceSourceType = Literal["vision", "dom", "api_external", "rule_deterministic"]`
  - Pas de fallback `llm_classifier` (interdit comme unique source — peut être combiné mais pas seul)
  - Validator : `source_type ∈ EvidenceSourceType`
- [ ] Migration backward : si un evidence_ledger.json existant a `source_type="llm_classifier"`, ne pas crasher mais logger warning + treat as invalid source pour gate (effectivement bloque le ship)

### Gate module

- [ ] Fichier `moteur_gsg/core/claims_source_gate.py` créé (mono-concern axe `validation`, ≤300 LOC — split si dépasse)
- [ ] Exposition fonction `validate_claims_sources(html: str, evidence_ledger: EvidenceLedger) -> ClaimsSourceReport` (Pydantic v2)
- [ ] Modèle `ClaimsSourceReport` :
  - `claims_found: list[Claim]` (chaque Claim = `selector`, `text_excerpt`, `evidence_id`, `is_valid`)
  - `claims_total: int`
  - `claims_with_valid_source: int`
  - `claims_missing_source: list[Claim]`
  - `claims_invalid_source: list[Claim]` (evidence_id exists mais source_type invalid)
  - `gate_passed: bool` (= len(missing) + len(invalid) == 0)
- [ ] Pattern HTML détection (BeautifulSoup ou regex strict) :
  - `<span class="number">` ou `<span class="stat">` ou `<strong class="claim">`
  - `<li class="proof-strip">` ou `<li class="proof">` ou `<li class="trust-indicator">`
  - `<blockquote class="testimonial">` ou `<div class="testimonial-card">` ou élément avec `data-testimonial`
  - `<img class="logo-strip">` ou `<img class="trust-logo">` ou élément avec `data-logo`
  - Texte contenant pourcentages, chiffres ronds (>10), monnaies, ratings (X/5, X stars)
- [ ] Pour chaque claim détecté : extraire `data-evidence-id` attribute, valider vs `evidence_ledger.json`
- [ ] Custom exception `ClaimsSourceError(report: ClaimsSourceReport)` raise quand `gate_passed=False`

### Wire mode_1

- [ ] Modifier `moteur_gsg/modes/mode_1_complete.py` :
  - Wire `validate_claims_sources()` **post-impeccable_qa, pre-multi-judge**
  - Si `gate_passed=False` :
    - En mode `--ship-strict` : raise `ClaimsSourceError`
    - En mode normal : log warning + tag GSG output avec `claims_source_gate_failed=True` (n'empêche pas la prod du HTML mais signale)

### Tests

- [ ] Tests pytest `tests/gsg/test_claims_source_gate.py` :
  - Fixture HTML avec testimonial avec `data-evidence-id` valide → gate_passed=True
  - Fixture HTML avec testimonial sans `data-evidence-id` → gate_passed=False, `ClaimsSourceError` raise en strict
  - Fixture HTML avec testimonial pointant vers evidence_id `source_type="llm_classifier"` → gate_passed=False
  - Fixture HTML avec chiffre 78% sans source → gate_passed=False
  - Fixture HTML "neutre" (intro générique, pas de claims) → gate_passed=True (rien à valider)
