# Task: Opportunity CLI + wire recos orchestrator

## Description

Finir l'Opportunity Layer en deux volets :

1. **CLI** : exposer `python3 -m growthcro.opportunities.cli prepare --client <slug> --page <page_type>` qui consomme l'orchestrator (ISSUE-48) et écrit `opportunities.json`.

2. **Wire recos** : modifier `growthcro/recos/orchestrator.py` pour qu'il **consomme `opportunities.json`** (et non plus les scores bruts), et ajoute un champ `linked_opportunity_id` sur chaque reco générée. Acceptance critique : **pas de régression en richesse** des recos (taille `recos_v13_final.json` constante).

Source : `AUDIT_DECISION_DOSSIER_2026-05-16` §6 P0.3 + §7 ISSUE-P0-05.

## Acceptance Criteria

### CLI

- [ ] Fichier `growthcro/opportunities/cli.py` créé (mono-concern axe `CLI`, ≤150 LOC)
- [ ] Commandes :
  - `prepare --client <slug> --page <page_type>` : génère opportunities.json pour 1 page
  - `prepare --client <slug>` : génère opportunities.json pour TOUTES les pages du client
  - `prepare --all` : génère opportunities.json pour TOUS les clients curatés (avec progress bar)
- [ ] Output JSON-line vers stdout pour les logs (logger structuré), résumé human-readable à la fin
- [ ] Exit code 0 si OK, 1 si erreur métier (scores manquants, doctrine invalide), 2 si erreur interne

### Wire recos orchestrator

- [ ] Modifier `growthcro/recos/orchestrator.py` :
  - Lire `opportunities.json` en input (ajout, pas remplacement)
  - Pour chaque opportunity, générer 1-N recos liées (existing logic)
  - Chaque reco produite reçoit un champ `linked_opportunity_id: str | None`
  - Si aucune opportunity ne match un critère (cas legacy), reco continue d'être générée sans `linked_opportunity_id` (backward compat)
- [ ] Modifier `growthcro/recos/schema.py` (ou modèle Pydantic correspondant) pour ajouter champ optionnel `linked_opportunity_id: str | None = None`
- [ ] Re-export type via `growthcro/models/recos_models.py` (cf. CODE_DOCTRINE §TYPING)
- [ ] Smoke test Weglot : `python3 -m growthcro.recos.cli enrich --client weglot --page home --dry-run`
  - Avant : taille recos_v13_final.json baseline
  - Après : taille ≥ baseline ET champ `linked_opportunity_id` populated sur ≥80% des recos
- [ ] Tests pytest `tests/recos/test_orchestrator_opportunities_wiring.py` :
  - Recos avec opportunities → linked_opportunity_id populated
  - Recos sans opportunities (legacy) → linked_opportunity_id is None, no error
