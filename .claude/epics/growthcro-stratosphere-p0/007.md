**Critères décision KEEP/PIN/DROP** :
- KEEP : skill activement invoqué (Python ou Skill tool), audit clean.
- PIN : dormant mais utilité future identifiée par Mathis (commit SHA pinned pour reproductibilité).
- DROP : dormant + remplaçable + pas de plan d'utilisation 6 mois.

**Méthodologie audit** (par skill) :
1. Lire `SKILL.md` complet
2. Lire tous les scripts/fichiers du dossier skill
3. Grep : `subprocess|os.system|eval|exec|urllib|httpx|requests|os.environ|os.getenv|git\.|gh `
4. Note Y/N + détail dans checklist
5. Décision KEEP/PIN/DROP avec justification
