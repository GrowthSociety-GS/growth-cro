# Task: Fix SSRF crawler validation

## Description

Le crawler Playwright accepte aujourd'hui n'importe quelle URL utilisateur sans validation. `growthcro/capture/orchestrator.py:68` appelle `await page.goto(url, ...)` directement, et `growthcro/capture/cli.py:20` passe `--url` brut. Le repo étant PUBLIC sur GitHub, un attaquant (fork, PR contributor, consultant agence avec accès) peut déclencher des scrapes vers `http://localhost:9000`, `http://169.254.169.254/latest/meta-data/` (AWS metadata service), `file:///etc/passwd`, ou n'importe quelle IP privée (10.0.0.0/8, 192.168.0.0/16, 172.16.0.0/12, 127.0.0.0/8, 169.254.0.0/16). Le HTML scraped est ensuite stocké dans `data/captures/<client>/...`, exposant éventuellement les données internes.

Créer un module mono-concern `growthcro/capture/url_validator.py` qui valide chaque URL avant `page.goto` et raise `URLValidationError` si dangereuse.

Source : `AUDIT_DECISION_DOSSIER_2026-05-16` §10.1 Top 5 risques sécurité (#1 CRITIQUE).
