# Task: Opportunity engine orchestrator (deterministic)

## Description

Implémenter l'**orchestrator déterministe** qui transforme `score_page_type.json + score_applicability_overlay.json + brief + evidence_ledger.json` en `opportunities.json`. Aucun appel LLM (au premier shot — LLM enrichment = issue P1 séparée).

Règles métier :
- Pour chaque critère du playbook V3.2.1 avec `score < weight * 0.8` → générer une `Opportunity`.
- Si une `applicability rule` a tiré sur cette page (ex: `rule_saas_coherence_required`), boost les opportunités du pilier concerné en `severity=critical` ou `priority_score` × 1.5.
- `impact` heuristique basé sur `weight × (target - current)`.
- `effort` heuristique basé sur `category` (copy=2, structure=3, design=4, dev=4, mixed=5).
- `confidence` = `min(applicability_rule_confidence, score_confidence)` du critère.
- `hypothesis` template : "If we fix `<criterion>` from `<current>` to `<target>`, we expect `<metric>` to improve by `<estimated_lift>` because `<reason from doctrine>`."
- `metric_to_track` mappé par catégorie (cf. dict statique).
- `evidence_ids[]` populated depuis `evidence_ledger.json` entries qui matchent le critère.

Source : `AUDIT_DECISION_DOSSIER_2026-05-16` §6 P0.3 + §7 ISSUE-P0-04 + l'agentic doc §9.
