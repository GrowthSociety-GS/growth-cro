# Task: `Opportunity` Pydantic schema + persistence

## Description

Créer la **fondation** de l'Opportunity Layer : un schema Pydantic v2 strict pour modéliser une "Opportunity" CRO + un module de persistence pour read/write `data/captures/<client>/<page>/opportunities.json`.

Pas d'orchestration ni de génération ici — juste le schema + I/O. L'orchestrator vient en ISSUE-48.

Source : `AUDIT_DECISION_DOSSIER_2026-05-16` §6 P0.3 + §7 ISSUE-P0-03 + l'agentic doc §9 Opportunity model.

## Acceptance Criteria

- [ ] Package `growthcro/opportunities/` créé avec `__init__.py` (package marker)
- [ ] Fichier `growthcro/opportunities/schema.py` créé (mono-concern axe `validation`, ≤200 LOC)
- [ ] Modèle Pydantic v2 `Opportunity` exposé avec champs :
  - `id: str` (slug-format, validator)
  - `criterion_ref: str` (ex: "hero_01", "persuasion_05" — lien vers playbook V3.2.1)
  - `page_type: str`
  - `client_slug: str`
  - `current_score: float` (0-100, validator)
  - `target_score: float` (0-100, validator, > current_score)
  - `impact: int` (1-5)
  - `effort: int` (1-5)
  - `confidence: int` (1-5)
  - `severity: Literal["low", "medium", "high", "critical"]`
  - `priority_score: float` (computed via validator : `(impact * confidence * severity_weight) / effort`, severity_weight = {low:1, medium:2, high:3, critical:5})
  - `category: Literal["copy", "design", "structure", "trust", "offer", "friction", "tech", "tracking", "mobile", "a11y"]`
  - `problem: str` (max 280 chars, what's broken)
  - `evidence_ids: list[str]` (Field(..., min_length=1), refs into evidence_ledger.json — V26.A invariant cf. CODE_DOCTRINE §TYPING)
  - `hypothesis: str` (max 280 chars, "If X then Y because Z")
  - `recommended_action: str` (max 500 chars)
  - `metric_to_track: str` (ex: "hero_cta_click_through_rate", "form_completion_rate", "bounce_rate")
  - `owner: Literal["copy", "design", "dev", "growth", "analytics", "mixed"]`
  - `created_at: datetime`
- [ ] `model_config = ConfigDict(extra="forbid", frozen=True)`
- [ ] Modèle `OpportunityBatch(BaseModel)` qui wrappe `opportunities: list[Opportunity]` + `client_slug` + `page_type` + `doctrine_version` + `created_at`
- [ ] Fichier `growthcro/opportunities/persist.py` créé (mono-concern axe `persistence`, ≤150 LOC) :
  - `load_opportunities(client_slug, page_type) -> OpportunityBatch | None`
  - `save_opportunities(batch: OpportunityBatch) -> Path` (write to `data/captures/<client_slug>/<page_type>/opportunities.json`, atomic write via tmpfile + rename)
- [ ] Tests pytest `tests/opportunities/test_schema.py` + `tests/opportunities/test_persist.py` :
  - Valid Opportunity passe validation
  - Invalid : `evidence_ids=[]` → ValidationError (V26.A invariant)
  - Invalid : `target_score <= current_score` → ValidationError
  - Invalid : `impact=0` ou `impact=6` → ValidationError (range 1-5)
  - Round-trip save/load preserves data
  - Atomic write : pas de fichier partiel en cas de crash mid-write
- [ ] Aucun appel LLM, aucune génération — schema + I/O purs

- Rationale : `Opportunity` est consommé par opportunities/orchestrator et recos/orchestrator. C'est un modèle inter-module → **devrait** théoriquement aller dans `growthcro/models/opportunity_models.py` per CODE_DOCTRINE §TYPING.

**Correction acceptance** : `schema.py` devient un thin re-export, le vrai modèle est dans `growthcro/models/opportunity_models.py`.
