**Décisions notables (agent worktree)**
- Encryption : pgcrypto `encrypt(token, key, 'aes')` with session GUC `app.reality_token_key` set by the API from `REALITY_TOKEN_ENCRYPTION_KEY` env var ; sentinel `__no_key__` triggers refusal-to-write — no plaintext path possible
- OAuth state validation : HMAC-SHA256 of `<slug>:<connector>` with `REALITY_OAUTH_STATE_SECRET`, constant-time `crypto.timingSafeEqual` ; mismatch redirects to `?error=invalid_state`
- REST not SDK : stdlib `urllib.request` (Python) + native `fetch` (TS), zero new npm/pip dependencies
- Cron schedule `0 * * * *` (hourly) — Vercel Cron rate-limit compliant, inserts a `runs` row that the Python worker daemon dispatches
- Basename disambiguation : V30 OAuth connectors use `_v30` suffix to avoid duplicating legacy V26.C basenames in `growthcro/reality/`
- Defensive everywhere : connector → `not_connected` when access_token missing ; callback → 503 `connector_not_configured` when env vars unset ; cron → 401 without `Authorization: Bearer $CRON_SECRET`
