# Task CR-01 — Creative Exploration Engine

## Description

Créer le package `moteur_gsg/creative_engine/` qui appelle Claude Opus 4.7 pour générer 3-5 directions créatives audacieuses pour une LP donnée. Sortie structurée Pydantic v2, pas de production HTML.

Source : PRD §FR-1 + Codex addendum §"Proposed New Layer: Creative Exploration Engine".

## Acceptance Criteria

- [ ] Package `moteur_gsg/creative_engine/` créé (mono-concern, 4 axes : validation + orchestration + persistence + CLI)
- [ ] `growthcro/models/creative_models.py` NEW : `CreativeRoute`, `CreativeRouteBatch`, `RouteThesis` (Pydantic v2 strict frozen+extra=forbid, ≤200 LOC)
  - `CreativeRoute` fields : `route_id`, `route_name`, `aesthetic_thesis`, `spatial_layout_thesis`, `hero_mechanism`, `section_rhythm`, `visual_metaphor`, `motion_language`, `texture_language`, `image_asset_strategy`, `typography_strategy`, `color_strategy`, `proof_visualization_strategy`, `page_type_modules: list[str]`, `risks: list[str]`, `why_this_route_fits: str`

**Anti-mega-prompt** : le system prompt Opus DOIT être <8K chars (CLAUDE.md #1). Structure :
```
Section 1 : context (brief + brand_dna + design_grammar) — max 4K chars
Section 2 : task spec (génère 3-5 routes JSON valid) — max 1K chars
Section 3 : output schema (JSON example) — max 2K chars
Section 4 : constraints (anti-AI-slop banned defaults, mais toolbox open) — max 1K chars
Total : ≤8K chars hard limit, validate at module load via assert
```

**Model** : `claude-opus-4-7` (cf. décisions tranchées epic.md). Pas de fallback Sonnet (Sonnet est pour copy, pas exploration).

**Coût attendu** : ~$0.5-1 par run de 5 routes (input ~5K tokens, output ~8K tokens × 5 routes).
