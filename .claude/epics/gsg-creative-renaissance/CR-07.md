## Description

Setup Promptfoo + golden set 6+ clients tests dans 6+ verticals. CI green = non-régression prouvée (composite, visual_ambition, brand_fit, time-to-LP, cost).

Source : PRD §FR-7.

- [ ] Dossier `_eval/` créé à la racine projet
- [ ] `_eval/renaissance_golden_set.yaml` Promptfoo config :
  - 6+ test cases (1 par vertical) — clients réels du fleet 56-clients :
    - Weglot (SaaS B2B) — baseline existing

## Technical Details

**Fichiers créés (~10+)** :
- `_eval/renaissance_golden_set.yaml` (~200 LOC YAML)
- `_eval/README.md` (~100 LOC)
- `_eval/fixtures/<client>/brief_v2.json` × 6 clients (copy from existing data/_briefs_v2/ + adapt)
- `_eval/fixtures/<client>/golden_screenshots/{desktop,mobile}.png` × 6 (reference snapshots when CR-06 stable)
- `_eval/scripts/run_gsg_for_eval.py` (~150 LOC) — wrapper script Promptfoo calls
- `tests/eval/test_eval_config.py` (~80 LOC)
- `package.json` updated avec `scripts.eval:renaissance`

**Coût per CI run** : 6 clients × ~$2/LP = ~$12 par CI run. Acceptable pour gate avant release.

**Promptfoo setup** :
- Install : `npm install -g promptfoo` (or via `npx promptfoo`)
- Config : `promptfoo init` then customize
- Run : `npx promptfoo eval -c _eval/renaissance_golden_set.yaml`
- UI : `npx promptfoo view` (local web UI for results browsing)
