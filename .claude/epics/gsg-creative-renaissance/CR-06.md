# Task CR-06 — Renderer extension multi-vertical

## Description

Étendre le renderer existant (`moteur_gsg/core/page_renderer_orchestrator.py` + modules connexes) pour qu'il consomme un `VisualComposerContract` (CR-03/CR-04) au lieu de juste `copy_doc`. Produit du HTML/CSS/JS multi-vertical avec image embed depuis DALL-E + motion CSS/JS depuis motion_specs.

Source : PRD §FR-6.

## Acceptance Criteria

- [ ] `moteur_gsg/core/page_renderer_orchestrator.py` étendu (split obligatoire si dépasse 500 LOC) :
  - New entry point `render_with_composer_contract(composer: VisualComposerContract, copy_doc: dict, brief: dict) -> RenderedPage`
  - Legacy entry point `render(copy_doc: dict, brief: dict)` continue de marcher (backward compat)
  - Si `composer` fourni → use modules + motion + assets ; sinon → use legacy rendering
- [ ] Per-vertical CSS modules dans `moteur_gsg/visual_composer/vocabulary/*.json` (CR-04 created) consommés via composer
- [ ] Image embed :
  - Pour chaque `AssetSpec` dans composer, call `growthcro.lib.image_client.generate_image()` (CR-04)
  - Embed `<img>` tag avec src pointing to local `data/captures/<client>/<page>/assets/<asset_id>.png` (relative ou absolute selon contexte)
  - Lazy loading + responsive `srcset` si plusieurs sizes
- [ ] Motion injection :
  - Pour chaque `MotionSpec`, generate CSS `@keyframes` + class name
  - Inject `<style>` block dans `<head>` avec toutes les keyframes
  - Apply classes aux target_selectors HTML
- [ ] Tests pytest :
  - `test_renderer_composer_path.py` : fixture VisualComposerContract minimal (1 section, 1 module, 1 motion, 0 assets) → HTML contient les bons selectors + CSS keyframes
  - `test_renderer_backward_compat.py` : appel render() sans composer → HTML identique au baseline current Wave 3
  - Smoke fixture Weglot lp_listicle : full Composer contract → HTML rendered passes ClaimsSourceGate + VerdictGate (no regression gates Wave 3)
