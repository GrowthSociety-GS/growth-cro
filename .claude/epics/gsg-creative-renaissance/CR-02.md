# Task CR-02 — Visual Judge / Route Selection

## Description

Module qui consume `CreativeRouteBatch` (CR-01 output) + brief + brand_dna, score chaque route sur 5 axes (1-10), sélectionne la best route avec rationale.

Source : PRD §FR-2 + Codex addendum §"Visual Judge / Route Selection".

## Acceptance Criteria

- [ ] Module `moteur_gsg/creative_engine/judge.py` créé (axe validation, ≤200 LOC)
- [ ] Modèle Pydantic `RouteScore` : `brand_fit (1-10)`, `cro_fit (1-10)`, `originality (1-10)`, `feasibility (1-10)`, `visual_potential (1-10)`, `weighted_score (computed)`, `rationale (min_length=50)`
- [ ] Modèle Pydantic `RouteSelectionDecision` : `selected_route_id`, `selected_route_score: RouteScore`, `alternatives_evaluated: list[RouteScore]`, `selection_rationale (min_length=100)`, `decision_timestamp`
- [ ] Fonction `select_route(batch: CreativeRouteBatch, brief: dict, brand_dna: dict) -> RouteSelectionDecision`
- [ ] Scoring : Sonnet 4.6 (cheaper que Opus, suffit pour scoring) + structured output via tool_use
- [ ] Weights default : `brand_fit=0.30, cro_fit=0.25, originality=0.15, feasibility=0.15, visual_potential=0.15` (configurable via constants)
- [ ] Persist `data/captures/<client>/<page_type>/route_selection_decision.json` (atomic, même pattern)
- [ ] Tests pytest :
  - `test_judge.py` : fixture batch 5 routes → 1 selected + 4 alternatives, scores valides
  - Score weighting : route avec brand_fit=10 + autres axes=5 → weighted ~6.25, doit être préférée si autres routes ont weighted <6.25
  - Tie-break : si 2 routes ont même weighted_score, prend celle avec brand_fit le plus haut

## Technical Details

**Fichiers créés (~3)** :
- `moteur_gsg/creative_engine/judge.py` (~180 LOC)
- `tests/creative_engine/test_judge.py` (~120 LOC)
- Reuse `growthcro/models/creative_models.py` (CR-01) pour les models

**Coût** : ~$0.1 par batch (Sonnet evaluating 5 routes structured ~3K input tokens × 5 + ~500 output tokens).

**Anti-pattern** : pas de mega-prompt judge. Le system prompt judge ≤4K chars (assertion au module load).

## Stop Conditions

13 standard. Particuliers :
- Si Sonnet output not parseable → retry once, sinon raise (pas de fake score)
- Si tous les 5 scores sont identiques sur tous axes → log warning, prend route_id alphabétique (fallback deterministic)
