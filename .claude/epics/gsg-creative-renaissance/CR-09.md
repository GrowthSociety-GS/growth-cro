# Task CR-09 — Elite Mode (Opus Unleashed direct-to-HTML)

## Description

Ajouter un **second chemin créatif** au GSG en complément du structured mode (Wave 1 CR-01/02/03). Elite Mode = Opus 4.7 produit **1-3 candidats HTML COMPLETS directement** (pas thesis abstraites). Cible : atteindre ou dépasser le niveau visuel `landing_page_gpt_max_demo.html` (GPT-5 single-shot ORBITAL démo) qui a motivé l'epic.

**Le mode structured Wave 1 N'EST PAS supprimé.** Il devient le mode `fallback` / `learning` / `stable production scaling 56 clients`. Elite Mode = mode `wow factor` / `client demo` / `benchmark`.

Source : Codex addendum 2026-05-16 + Codex review pivot 2026-05-17 (4 corrections inscrites ci-dessous).

## 🔒 Constraint Statement Codex (verbatim — NON NÉGOCIABLE)

> 1. Elite HTML candidates are **NOT converted** to VisualComposerContract.
> 2. Elite output **must preserve** original layout/CSS/motion unless a **deterministic gate** finds a **concrete blocking issue**.
> 3. Renderer (CR-06) is **fallback/structured path ONLY**.
> 4. Convergence between structured and elite modes happens at **post-process gates** (evidence/claims/SEO/screenshots/multi-judge/persist), **NEVER at rendering layer**.

Si un quelconque test ou design suggère de violer ces 4 contraintes → **STOP** + ask Mathis avant de continuer.

### Orchestrator (Opus Unleashed)

- [ ] `orchestrator.py::generate_html_candidates(brief, brand_dna, page_type, business_category, *, n_candidates=3, references=None) -> HtmlCandidateBatch` :
  - Model : `claude-opus-4-7` (à valider live, fallback `claude-opus-4-1` si 404)
  - `max_tokens=24000` par candidate (Opus output HTML complet peut atteindre 15-25K tokens)
  - `temperature=0.85` (créatif, pas 0.1 production)
  - **PAS de tool_use forced JSON** (HTML libre, parsing via extract de premier `<!DOCTYPE html>` ou `<html>` block)
  - 1 call Opus génère 1 candidate, **boucle pour 3 candidates** (3 calls indépendants pour diversité — sinon Opus va répéter même approche)
  - System prompt FOCALISÉ ≤6K chars (assertion module load anti-pattern CLAUDE.md #1) :
    - **Section 1 (~1.5K)** : Role + permission créative explicite ("you have full creative latitude on layout, motion, hierarchy, texture, decorative systems, hero mechanics — produce a single-file HTML LP that would make a senior art director proud")
    - **Section 2 (~1K)** : Creative Bar criteria (per-vertical depuis `creative_bar.py`)
    - **Section 3 (~1.5K)** : Brand DNA constraints (couleurs hex, typos, spacing tokens, voice/tone, **forbidden_visual_patterns** depuis brief V2)
    - **Section 4 (~1K)** : Toolbox autorisée vs bannie (cf. Codex addendum §"Anti-AI-slop rules became too defensive" — autorise gradients, glass, motion, depth, dashboard mockups, conic logos, marquee SVG ; bannit stock photos, generic gradient mesh, fake icons, AI slop)
    - **Section 5 (~1K)** : Hard constraints output (single-file HTML autonome, CSS inline `<style>`, JS inline `<script>`, mobile-first responsive, ≥800 LOC, ≤4000 LOC, includes `<title>` ≤60 chars + `<meta description>` ≤160 chars, claim numbers/testimonials/logos avec `data-evidence-id` attributes si possible mapping avec brief.sourced_numbers)
  - User message ≤4K chars : brief.objective + audience + angle + traffic_source + page_type + business_category + brand_dna summary
  - **AUCUNE référence HTML golden injectée par défaut** (anti-mimicry, cf. Codex correction #4)
  - Retry once si HTML output unparseable (no `<!DOCTYPE>` or `<html>` block found) — call back to Opus avec error feedback
  - Si second retry fails → `raise EliteCreativeError("Opus output not valid HTML after retry", upstream=...)`
  - Logger structured : `logger.info("elite.candidate_generated", extra={"client": ..., "candidate_id": ..., "html_chars": ..., "cost_usd": ..., "wall_seconds": ...})`
