# Task CR-04 — Visual Composer (image gen SKIP v1)

## Description

Package `moteur_gsg/visual_composer/` qui transforme `CreativeRouteContract` (CR-03) + vocabulary library multi-vertical → `VisualComposerContract` concret (sections + modules + motion + asset specs). Resolver d'assets utilise **Brand DNA per-client existing assets + SVG vocabulary library + Pexels/Unsplash stock fallback** (gratuit, no auth). **PAS d'image gen IA en v1** per décision Mathis 2026-05-16.

Source : PRD §FR-4 + Codex addendum §"Visual Composer".

## Acceptance Criteria

- [ ] Package `moteur_gsg/visual_composer/` créé
- [ ] Vocabulary library JSON `moteur_gsg/visual_composer/vocabulary/` :
  - `saas.json` (B2B SaaS pattern : dashboard mockups SVG, founder-led narrative, ROI graphs)
  - `ecommerce.json` (PDP-style, product cards, urgency, badges)
  - `luxury.json` (minimal, large negative space, serif typography, B/W photography hints)
  - `marketplace.json` (multi-supply, social proof high density, search-first hero)
  - `app_acquisition.json` (mobile-first, app store badges, video previews)
  - `media_editorial.json` (long-form, editorial typography, byline, citations)
  - `local_service.json` (trust signals, contact prominent, before-after, testimonials geo)
  - `health_wellness.json` (calming palette, science-backed proof, transformations)
  - `finance.json` (trust + compliance, numbers prominent, security badges)
  - `creator_course.json` (instructor-led, curriculum preview, social proof learners)
  - `enterprise.json` (logos tier 1, security/SOC2 prominent, case studies depth)
  - `consumer_brand.json` (lifestyle, emotional, large hero imagery)
  - Each file : `hero_mechanisms[]`, `section_rhythms[]`, `visual_metaphors[]`, `motion_patterns[]`, `texture_rules[]`, `asset_strategies[]` (SVG patterns, gradients CSS specs, Lottie animation refs, brand DNA asset references), `typography_scales[]` (5-15 items per category)
- [ ] `moteur_gsg/visual_composer/composer.py` (axe orchestration, ≤250 LOC) :
  - `compose(route_contract: CreativeRouteContract, business_category: str, brief: dict, brand_dna: dict) -> VisualComposerContract`
  - Lookup vocabulary[business_category]
  - Match route.thesis fields to vocabulary items (greedy + Sonnet fallback if ambiguous)
  - Generate concrete section specs + modules + motion + asset specs
- [ ] `moteur_gsg/visual_composer/asset_resolver.py` (axe orchestration, ≤200 LOC) :
  - Pour chaque `AssetSpec` (logo / hero_image / background / icon / illustration), resolver dans cet ordre :
    1. **Brand DNA per-client** : check `brand_dna.assets.<asset_type>` dans `data/captures/<client>/brand_dna.json` (existing logos, photos produits, etc.)
    2. **SVG vocabulary library** : check `moteur_gsg/visual_composer/vocabulary/<category>.json` `asset_strategies[]` pour SVG patterns inline
    3. **Pexels/Unsplash stock fallback** : si vraiment besoin photo stock, query Pexels API (no auth) avec keywords du route thesis. Lazy import, returns first relevant result URL
    4. Si rien match → AssetSpec marquée `unresolved`, le renderer va utiliser un fallback CSS gradient placeholder
  - Output : `AssetSpec` enrichi avec `resolved_path` ou `resolved_svg_inline` ou `resolved_url` ou `unresolved: True`
  - **PAS d'image gen IA** (skip v1)
- [ ] `moteur_gsg/visual_composer/motion_specs_generator.py` (axe I/O serialization, ≤120 LOC) :
  - Convertit `motion_language` thesis en `MotionSpec` concrets (CSS keyframes ou JS animation hints)
- [ ] Tests pytest :
  - `test_composer.py` : 3 routes différentes × 3 verticals → 9 VisualComposerContract valides
  - `test_asset_resolver.py` : 1 asset_spec → resolved depuis (a) Brand DNA OR (b) SVG library OR (c) Pexels stock OR (d) unresolved fallback (4 paths testés)
  - `test_motion_specs_generator.py` : 1 motion_language → ≥1 MotionSpec
  - Mock Pexels call (no real API hit in CI)
