# T15-2 — Source label leak fix + anti-invention guard renforcé

**Status** : 📝 DRAFT
**Priority** : 🔴 BLOCKING
**Estimated** : 45-60 min

## Why

Mathis 2026-05-15 (screen 2) : *"c'est quoi ce truc ? C'est moche et en
plus tu cites le module ou des infos de la webapp ? Personne comprend
on veut pas ça"*

Bug confirmé : `_proof_strip()` rend
`<li><span>...</span><small>home/capture.structure.headings</small></li>`.
Ces labels (`home/capture.structure.headings`, `brief`,
`recos_v13_final`) sont des chemins **internes** vers les artefacts
data, jamais censés être visibles à un end user.

Et (screen 3) : *"c'est les vrais infos ça ? Les vrais avis ? J'en
doute fortement"* — les testimonials affichés ("Équipe Growth ·
Polaar", etc.) viennent du BriefV2 fabriqué par Claude — aucune URL
source publique. La doctrine anti-invention doit les refuser.

## Acceptance criteria

1. **Source label render** : remplacer le format `<small>home/capture...</small>`
   par soit :
   - format public : `<small>Source · {client}.com — {YYYY-MM}</small>`
   - OU rien (masquer entièrement) si source non-mapable à un label
     publishable
2. **Impeccable QA rule `internal_provenance_leak`** : ajouter dans
   `moteur_gsg/core/impeccable_qa.py` (ou data ruleset) :
   - match regex `\b(home|pricing|lp_leadgen|pdp)/(capture|recos|score|perception)\.\w+`
   - match `<small>(brief|recos_v\d+|score_\w+|perception_v\d+)</small>`
   - severity = `block` (fail run)
3. **Anti-invention testimonials** : dans
   `moteur_gsg/modes/mode_1_complete.py` (ou `brief_v2.py`), ajouter
   validation :
   - chaque `testimonials[i].source_url` requis OU
     `testimonials[i].sourced_from = "internal_brief"`
   - si `source_url` absent ET `sourced_from != "internal_brief"` →
     renderer affiche `[non-vérifié]` overlay OU skip le card
4. **Fallback proof strip** : si plan a 0 facts publishable, rendre
   un single line cohérent avec la doctrine, pas du debug stuff.
