# T15-3 — LP-Creator copy preservation → injection moteur GSG

**Status** : 📝 DRAFT
**Priority** : 🔴 BLOCKING
**Estimated** : 60-75 min

## Why

Mathis 2026-05-15 : *"c'est passé où les 4 phases avec ta proposition
de rédaction, les mentions de Amazon, Microsoft etc.."*

Audit : LP-Creator V2 produit un copy structuré 20/20 (validé par
Mathis lors de la session). Mais le moteur GSG `mode_1_complete` **re-
génère intégralement le copy via Sonnet** à chaque run, en utilisant
uniquement les champs structurés du BriefV2 (`objectif`, `audience`,
`angle`, `concept_description` mais pas un copy.md validé).

Résultat : les phrases LP-Creator validées (mentionnant Amazon, HBO,
les 10 hooks signature) sont **perdues**.

## Acceptance criteria

1. BriefV2 schema étendu : nouveau champ
   `lp_creator_validated_copy_path: str | None` qui pointe vers un .md
   structuré (ex: `deliverables/gsg_demo/weglot-listicle-2026-05-15-COPY.md`).
2. Quand ce path est présent, `mode_1_complete` :
   - charge le .md
   - le parse en sections (hero / intro / reasons[0..N] / final_cta / footer)
   - injecte chaque section comme **input "polish"** dans le prompt
     Sonnet — pas comme input "from scratch"
3. Le prompt système Sonnet (`COPY_SYSTEM_PROMPT` dans
   `prompt_assembly.py`) est étendu :
   > "The LP-Creator copy in the user prompt is CANONICAL. You polish
   > for layout integration (length tuning, microcopy variants only)
   > — never rewrite substance. Preserve all named entities, numbers,
   > and signature phrases verbatim."
4. Test d'acceptance : si on injecte le copy LP-Creator Weglot 20/20
   dans le brief, la sortie HTML reproduit ≥ 90% des phrases verbatim
   (mesuré par `python3 scripts/diff_copy_preservation.py` — nouveau
   script à créer, simple Levenshtein/Jaccard).

## Files to modify

- `moteur_gsg/core/brief_v2.py` :
  - ajouter `lp_creator_validated_copy_path: Optional[str]` dans BriefV2 dataclass
  - validate path existe et fichier lisible si fourni
- `moteur_gsg/core/copy_lp_creator_parser.py` (nouveau, 80-120 LOC) :
  - parse markdown → dict {section_id: {heading, body, bullets, microcopy}}
  - support la convention LP-Creator (cf. weglot-listicle-2026-05-15-COPY.md)
- `moteur_gsg/core/copy/prompt_assembly.py` :
  - si `plan.lp_creator_copy` présent, étendre prompt avec :
    ```
    ## LP-CREATOR CANONICAL COPY (PRESERVE VERBATIM)
    {parsed_copy_json}
    ```
  - étendre `COPY_SYSTEM_PROMPT` avec la règle "preserve verbatim"
- `moteur_gsg/modes/mode_1_complete.py` :
  - charger et parser le copy LP-Creator avant `build_copy_prompt()`
- `scripts/diff_copy_preservation.py` (nouveau) :
  - input : copy.md original + HTML rendu
  - output : % de phrases LP-Creator présentes dans HTML
