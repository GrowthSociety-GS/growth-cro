# T15-5 — Wire `cro-methodology` skill runtime

**Status** : 📝 DRAFT
**Priority** : 🟡 POLISH (post-BLOCKING)
**Estimated** : 45-60 min

## Why

Sprint 14 audit honnête : seul `impeccable_qa` est wired au runtime du
moteur GSG. Les 4 autres skills installés (`frontend-design`,
`brand-guidelines`, `emil-design-eng`, `cro-methodology`) sont des
knowledge bases dev-time, jamais invoquées pendant `generate_lp()`.

T15-5 wire **un** skill au runtime — `cro-methodology` car c'est celui
qui ROI le mieux en post-process copy (les 30 critères CRO sont
mesurables et actionnables, contrairement aux skills purement design).

Cost-benefit estimé : +30-60s wall + $0.10 par run, en échange d'un
score CRO mesurable + critiques actionnables pour retry-on-fail.

## Acceptance criteria

1. Après `apply_minimal_postprocess(html)` et avant
   `run_multi_judge(...)` (dans `mode_1_complete.py`), invoquer
   `Skill('cro-methodology', args='audit')` avec :
   - input : `copy_doc` JSON + `brief.objective` + `brief.audience` +
     `plan.page_type`
   - output attendu : JSON `{score: 0-100, critiques: [str], hits: [...]}`
2. Le résultat est sauvegardé dans
   `<output_dir>/cro_methodology_audit.json`
3. Si `score < 70` ET `retry_on_low_cro=True` (config) :
   - relancer un Sonnet copy call avec les critiques injectées dans
     le system prompt comme contraintes
   - max 1 retry pour éviter spirale de coût
4. Le score CRO est mergé dans `canonical_run_summary.json` à côté de
   `multi_judge.final_score`
5. Acceptance test : Weglot V8 run produit `cro_methodology_audit.json`
   avec un score ≥ 70

## Files to modify

- `moteur_gsg/modes/mode_1_complete.py` :
  - import + helper `_invoke_cro_methodology_audit(copy_doc, brief, plan)`
  - hook dans le pipeline entre postprocess et multi-judge
  - merge result dans summary
- `moteur_gsg/core/cro_methodology_wrapper.py` (nouveau, ~80 LOC) :
  - wrapper pur autour de `Skill('cro-methodology', ...)`
  - retry/timeout handling
- `growthcro/config.py` :
  - flag `GSG_CRO_METHODOLOGY_RETRY: bool = False` (par défaut off)
