# T15-4 — Tier-1 logos grid render

**Status** : 📝 DRAFT
**Priority** : 🟡 POLISH (post-BLOCKING)
**Estimated** : 30-45 min

## Why

Le brief V2 Weglot liste `available_proofs: ['logos_clients_tier1']` —
HBO, Nielsen, IBM, Décathlon, Roche, Amazon. **Aucun renderer** ne
consomme actuellement ce flag : grep `logos_clients_tier1` dans
`moteur_gsg/core/` → 0 résultats hors validation enum.

Mathis 2026-05-15 : *"c'est passé où… les mentions de Amazon, Microsoft
etc.."* — il s'attendait à voir ces logos dans le hero.

## Acceptance criteria

1. Quand `brief.available_proofs.includes('logos_clients_tier1')` :
   - chercher la source des noms dans cet ordre :
     a. `brief.client_logos_tier1[]` (nouveau champ string list)
     b. `data/captures/<client>/home/clients_mentioned.json` (nouveau
        artefact — extracteur séparé)
     c. fallback : skip (ne pas inventer)
2. Renderer un `<ul class="logos-grid">` avec wordmarks **sobres** :
   - texte (pas SVG inline pour éviter de devoir loader 8+ SVGs)
   - police monospace ou display, opacity 0.5, hover 1.0
   - label "Ils nous font confiance" / "Trusted by" selon target_language
3. Position : dans le hero après le CTA primary, AVANT la proof-strip
4. Limit : max 6 logos affichés, ordre = ordre du brief
5. Mobile : stack en 2 colonnes (max 4 logos visibles)

## Files to modify

- `moteur_gsg/core/brief_v2.py` : ajouter `client_logos_tier1: list[str]`
- `moteur_gsg/core/page_renderer_orchestrator.py` :
  - nouveau helper `_render_logos_grid(plan)` au-dessus de `_hero_visual`
  - appel dans le hero après `cta-button`
- `moteur_gsg/core/section_renderer.py` :
  - équivalent pour les pages non-listicle (advertorial, lp_sales)
- `moteur_gsg/core/css/components.py` :
  - rules `.logos-grid`, `.logos-grid li`

## Out of scope

- Auto-extraction de logos depuis le HTML capturé du site source →
  ce serait T15-1 ou Sprint 16 (NLP simple sur `<img alt=""> <a href="">`
  patterns)
- SVG wordmarks vrais → Sprint 16 (besoin d'une biblio de SVGs)
