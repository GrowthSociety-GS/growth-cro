# T15-6 — Learning loop codifié `memory/SPRINT_LESSONS.md`

**Status** : 📝 DRAFT
**Priority** : 🟢 META (transverse — bloque le close de Sprint 15)
**Estimated** : 30 min initial + 10 min/sprint ongoing

## Why

Mathis 2026-05-15 : *"y'a une boucle d'apprentissage qui s'opère à
chaque fois depuis nos itérations et nos progressions ? On en tire des
règles des process des workflows etc ?"*

Réponse honnête : **non, pas formellement**. Chaque Sprint est commit
avec un message descriptif mais les leçons ne sont jamais distillées
en règles réutilisables, applicables automatiquement aux prochains
sprints. Conséquence prévisible : régressions identiques d'un sprint
à l'autre.

T15-6 initialise la boucle formelle.

## Acceptance criteria

1. Créer `memory/SPRINT_LESSONS.md` avec :
   - structure invariante : `## Sprint N — <date> — <title>` →
     `### Règles dégagées` (1-3 règles par sprint, format `règle |
     déclencheur | conséquence si violée`)
2. Initialiser avec les leçons de **Sprint 13 + 14** rétro-actives
   (au moins 7 règles déjà observées, cf. ci-dessous).
3. Référencer `memory/SPRINT_LESSONS.md` dans
   [`.claude/memory/MEMORY.md`](../../../memory/MEMORY.md) index.
4. Ajouter dans [`docs/doctrine/CODE_DOCTRINE.md`](../../../../docs/doctrine/CODE_DOCTRINE.md)
   section "Learning loop" : convention + obligation à la clôture
   d'un sprint.
5. (Stretch) ajouter un hook pre-commit qui rappelle de mettre à jour
   `SPRINT_LESSONS.md` si le commit est `feat(gsg):` ou `fix(gsg):`
   significatif.

## Initial seed — 7 règles déjà observées (Sprint 13 + 14)

| Règle | Déclencheur | Conséquence |
|------|-------------|-------------|
| Les screenshots du client doivent être **lang-aware** (cible FR → capture `/fr`, pas racine EN). | Capture client + génération multilingue | Hero incohérent visuellement → "écran de fumée" perçu |
| `<small>{internal_path}</small>` ne **doit jamais** atteindre le HTML rendu — c'est du debug, pas du contenu. | Renderer `_proof_strip`, `_fact_chips` | Mathis: "personne comprend on veut pas ça" |
| Les `testimonials.items[]` sans `source_url` publique sont **inventés** par défaut — refuser ou marquer `[non-vérifié]`. | Brief V2 testimonials + Sonnet copy generation | Risque légal + perte de confiance |
| Un copy LP-Creator validé (20/20 score Mathis) **ne doit jamais être écrasé** par re-génération Sonnet from scratch. | mode_1_complete copy stage | Perte du travail de qualification + des phrases signature |
| Un flag `available_proofs` dans le brief **doit avoir** un renderer correspondant. Sinon supprimer le flag du brief V2 enum. | Brief V2 schema vs renderer | Promesses non tenues |
| Le format `_reason_visual` doit produire des visuels **topiquement pertinents** au heading. SVG abstraits = AI-slop perçu. | component_renderer | "C'est moche / on comprend pas" |
| Chaque ajout de skill au runtime doit passer un **gate ROI** explicite : +cost +latency justifié par +qualité mesurable. | Skill wiring tickets | Cost creep silencieux |
