# T15-1 — Fresh capture + per-reason contextual visuals

**Status** : 📝 DRAFT
**Priority** : 🔴 BLOCKING
**Estimated** : 90-120 min

## Why

Mathis 2026-05-15 : *"je vois la capture du site en anglais avec une
description en anglais : j'espère que t'es pas allé chercher des images
existantes"* — la capture `desktop_clean_fold.png` actuelle est de
`weglot.com` (EN). Pour une LP cible `target_language=FR`, le hero
devrait afficher `weglot.com/fr`. Et chaque reason mérite un visuel
contextuel (dashboard, schema SEO, grid intégrations…) pas juste un
SVG icon générique.

## Acceptance criteria

1. Si `target_language=FR` ET le site source a une version `/fr` (détectée
   via discovered_pages.json), la capture utilisée est celle de `/fr` —
   pas la racine EN.
2. Si la capture existante est > 7 jours, trigger un Playwright recapture
   automatique (background, ne bloque pas le run).
3. Pour les pages SaaS B2B (`brief.business_category="saas"`), capturer en
   plus de la home : `dashboard|onboarding`, `pricing`, `integrations`,
   `customers|case-studies`. Stocker sous `data/captures/<client>/<deeplink>/screenshots/`.
4. `component_renderer._reason_visual()` étendu : avant de tomber sur le
   SVG icon, chercher un asset `<reason_topic>_screenshot.png` :
   - reason topic = "intégrations" → `integrations/screenshots/desktop_clean_fold.png`
   - reason topic = "5 minutes setup" → `onboarding/screenshots/desktop_clean_fold.png`
   - reason topic = "G2 reviews" → `customers/screenshots/desktop_clean_fold.png`
5. Au minimum 3 reasons sur 10 doivent avoir un visuel **non-générique**
   sur le run Weglot V8 d'acceptance.
