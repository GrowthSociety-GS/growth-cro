# Task: Split capture god files into growthcro/capture/

## Description

Break the two largest capture-side god files into a real package along the canonical concern axes (browser orchestration / cloud adapter / DOM serialization / persistence / orchestration / CLI). After the split, no file in the capture sub-tree exceeds 800 LOC and each file claims a single concern.

**Targets:**
- `ghost_capture_cloud.py` (1,122 LOC, repo root)
- `skills/site-capture/scripts/native_capture.py` (1,032 LOC)

Both move into `growthcro/capture/`. Old paths become **shim wrappers** that import-and-forward, so existing callers (sub-agents, shell scripts, docker-compose) keep working until Task 10 removes the shims.

## Acceptance Criteria

- [ ] `growthcro/capture/__init__.py` created.
- [ ] `growthcro/capture/` contains, at minimum:
  - `browser.py` — Playwright/Puppeteer orchestration
  - `cloud.py` — Apify/Browserless adapter
  - `dom.py` — DOM serialization, HTML cleanup
  - `persist.py` — write captures to `data/captures/<client>/<page>/`
  - `orchestrator.py` — the run-all loop, batch logic
  - `cli.py` — argparse entrypoint (was the `if __name__ == "__main__"` block)
- [ ] No file in `growthcro/capture/` exceeds 800 LOC.
- [ ] Every file in `growthcro/capture/` opens with a one-line module docstring naming its single concern (e.g. `"""Apify/Browserless cloud capture adapter — single concern: cloud delegation."""`).
- [ ] Old paths preserved as shim wrappers:
  - `ghost_capture_cloud.py` → 5–10 line shim that imports from `growthcro.capture` and forwards
  - `skills/site-capture/scripts/native_capture.py` → same
- [ ] All `os.environ`/`os.getenv` calls in the moved code replaced with `growthcro.config` (carry over from Task 002 — verify clean).

## Technical Details

- **Splitting axis** is concern, not lines. If after the natural split a single file still exceeds 800 LOC, break further by data shape (e.g. one persist module per artefact type).
- Imports follow the layering: `cli` → `orchestrator` → `browser`/`cloud`/`dom` → `persist`. No back-references.
- The shim pattern:
  ```python
  # ghost_capture_cloud.py — shim, will be removed in Task 10
  from growthcro.capture.cli import main  # noqa
  if __name__ == "__main__":
      main()
  ```
- Keep `git mv` history where possible; for a real split (1 file → N files), use `git mv` for the largest concern then `git add` the rest with a body-message that lists the split.
- Post-split: post the file map (which lines went where) as a comment on the GitHub issue **before** the commit lands, for Mathis review.
