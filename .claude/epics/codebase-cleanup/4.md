# Task: Move archives out of active paths

## Description

Archives must live exclusively under `_archive/`. Today, `skills/site-capture/scripts/_archive_deprecated_2026-04-19/` sits inside the active skills tree, and 18+ client capture directories under `data/captures/<client>/_obsolete_pages_2026-04-29/` pollute the live data folder. Sub-agents, the capabilities-keeper, and any LLM cold-reading the tree can mistake archived code for current. Relocate everything; preserve git history via `git mv`.

## Acceptance Criteria

- [ ] `skills/site-capture/scripts/_archive_deprecated_2026-04-19/` → `_archive/site-capture/2026-04-19/` (via `git mv`).
- [ ] Every `data/captures/*/_obsolete_pages_2026-04-29/` → `_archive/data_obsolete/<client>/2026-04-29/`.
- [ ] Every `data/captures/*/_obsolete_*` (any date) → same pattern.
- [ ] `find . -type d \( -name "_archive*" -o -name "_obsolete*" -o -name "*deprecated*" -o -name "*backup*" \) -not -path "./_archive/*" -not -path "*/node_modules/*" -not -path "*/worktrees/*"` returns nothing.
- [ ] `audit_capabilities.py` re-run; orphan count not increased vs Task-1 baseline (likely decreased, since archived files are no longer scanned as active).
- [ ] `parity_check.sh weglot` passes (no behavioural change — archives are unreferenced by definition).
- [ ] Single commit `Issue #<N>: move archives out of active paths`.

## Technical Details

- All moves are mechanical `git mv`. No content edits.
- If `audit_capabilities.py`'s scan paths hardcode `skills/site-capture/scripts/` and stop at depth N, verify it now correctly excludes the new `_archive/...` subtree. If broken, fix in this commit.
- Verify no active code imports from the archive paths via grep before moving:
  - `grep -r "_archive_deprecated" --include="*.py" . | grep -v _archive`
  - `grep -r "_obsolete_pages" --include="*.py" . | grep -v _archive`
  - If any active hit: stop, escalate to Mathis. (None expected — these are post-fact archive renames.)
- Document the move in `_archive/README.md` (append a date-stamped line per relocation block).
