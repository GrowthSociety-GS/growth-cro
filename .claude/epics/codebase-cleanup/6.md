# Task: Split recos god files & dedupe enricher variants

## Description

The recos generation is the worst single offender: `reco_enricher_v13.py` (1,690 LOC) is a four-concerns-in-one file (prompt assembly + API client + persistence + orchestration). It has duplicate copies in `scripts/` and `skills/site-capture/scripts/`, plus a sibling `reco_enricher_v13_api.py` (743 LOC) that overlaps with the API-call concern. Collapse into a clean `growthcro/recos/` package with one concern per file, single canonical entrypoint, single source of truth.

**Targets:**
- `skills/site-capture/scripts/reco_enricher_v13.py` (1,690 LOC, canonical)
- `skills/site-capture/scripts/reco_enricher_v13_api.py` (743 LOC)
- `scripts/reco_enricher_v13.py` (duplicate)
- `scripts/reco_enricher_v13_api.py` (duplicate)
- `scripts/enrich_v143_public.py` (1,339 LOC) — overlaps; verify if it's the public-API variant of the enricher and merge its concerns into the same package
