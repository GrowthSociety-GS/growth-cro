# Task: Centralize env/config in growthcro/config.py

## Description

Create the `growthcro/` package and its first module: `growthcro/config.py`. Single import surface for all environment variables. Replace every `os.environ[...]` and `os.getenv(...)` call in active paths. After this task, exactly one file in active paths reads env. This is the foundation every Wave-2 split builds on — doing it once now avoids touching every split twice.

## Acceptance Criteria

- [ ] `growthcro/__init__.py` created (empty or trivial).
- [ ] `growthcro/config.py` created with:
  - A `Config` class (singleton via `@functools.lru_cache` accessor `get_config()`)
  - Typed accessors: `anthropic_api_key()`, `apify_token()`, `openai_api_key()`, `perplexity_api_key()`, `browserless_token()`, `web_vitals_provider()`, etc. (one per known var)
  - `MissingConfigError(var: str, hint: str)` raised at startup-validation if a required var is unset
  - `.env` loaded via `python-dotenv` if present (already in `requirements.txt`? — check; if not, fall back to plain `os.environ` reading)
  - Optional vs required vars distinguished
- [ ] Every `os.environ[...]` and `os.getenv(...)` call in active paths replaced with the appropriate `config.*` call. Active paths = everything except `_archive/`, `node_modules/`, `__pycache__/`, `worktrees/`.
- [ ] `grep -rE "os\.environ|os\.getenv" --include="*.py" . | grep -v _archive | grep -v __pycache__ | grep -v node_modules | grep -v worktrees | grep -vE "(growthcro/)?config\.py"` returns nothing.
- [ ] `.env.example` regenerated from the config module's known vars (plus a comment line per var explaining its purpose).
- [ ] `parity_check.sh weglot` still passes byte-for-byte against the Task-1 baseline.
- [ ] `state.py` and `audit_capabilities.py` still run.

## Technical Details

- Known env vars seen in current code (from PRD inventory):
  - `ANTHROPIC_API_KEY` (required)
  - `APIFY_TOKEN` (required for cloud capture)
  - `OPENAI_API_KEY` (required for some judges)
  - `PERPLEXITY_API_KEY` (optional, GEO module)
  - `GROWTHCRO_WEB_VITALS_PROVIDER` (optional, default `"none"`)
  - `BROWSERLESS_TOKEN` (Docker only, optional default `"growthcro-local"`)
  - `BROWSER_WS_ENDPOINT` (Docker only, derived)
- The single Config singleton avoids re-reading `.env` per call. `get_config()` caches.
- Validation happens when an accessor is first called for a required var, not at import (so importing `config` is side-effect free).
- Error message pattern: `MissingConfigError("ANTHROPIC_API_KEY", "Set this in .env (see .env.example) or export it before running.")`.
- For Docker: `docker-compose.yml` already passes vars — no change there. Only the Python reads change.
- This task does NOT move `api_server.py` or any root file. Those stay where they are; they just import from `growthcro.config`. Root reorg is Task 8.
