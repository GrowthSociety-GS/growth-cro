## Technical Details

### Identification des 3 concerns

Read `moteur_gsg/core/copy_writer.py` end-to-end. Catégoriser chaque fonction/classe selon son concern :

1. **Prompt assembly** : tout ce qui construit le prompt string (f-strings, formats, injections de variables, normalization, escape).
2. **LLM API call** : tout ce qui parle au LLM (Anthropic client init, `messages.create(...)`, response handling, retries, error catching, token counting).
3. **Serializers** : tout ce qui transforme l'output LLM en JSON/dict (validation, evidence_id tagging, schema enforcement, file persistence).

Si une fonction est ambiguë → la garder dans le concern le plus dominant + commentaire explicatif. Si vraiment hybride → décomposer en 2 sous-fonctions, une par concern.
